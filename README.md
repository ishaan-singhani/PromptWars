## 📋 Problem Statement

The immense pressure of competitive exam preparation in India leads to widespread anxiety, burnout, and emotional fatigue among students — yet there is no accessible, student-specific tool to help them monitor and manage their mental wellness. **MindBoard** addresses this gap by providing a private, judgment-free daily wellness companion that helps students track their emotional state, understand their stress patterns, and receive personalized AI-powered support.

## ✨ Features

- **Daily Mood Tracking** — Log your emotional state each day with a simple, accessible check-in flow
- **Stress Trigger Identification** — Select from common academic stressors (study load, self-doubt, family pressure, etc.) to track patterns
- **Guided Breathing Exercise** — A visual Box Breathing tool for instant relief during acute stress moments
- **5-Minute Brain Dump Journal** — Private journaling space to offload worries and clear mental space before studying
- **AI-Powered Wellness Tips** — Personalized, empathetic reframe advice generated by Google Gemini based on your mood and exam context
- **Daily Streak System** — Build a healthy habit of self-reflection with an encouraging check-in streak tracker
- **Insights Dashboard** — Visualize your 7-day mood trends, identify your top stressor, and receive data-driven wellness insights

## 🚀 Setup Instructions

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Set up environment variables**
   ```bash
   cp .env.example .env
   ```
   Fill in the values in `.env` (see [Environment Variables](#-environment-variables) below).

3. **Start the development server**
   ```bash
   npm run dev
   ```
   The app will open at `http://localhost:5173` by default.

> **Note:** If Firebase credentials are not configured, MindBoard will automatically fall back to a local storage mock mode — all features remain functional for development and demo purposes.

## 🧪 Test Instructions

## 🔐 Environment Variables

Create a `.env` file in the project root based on `.env.example`:

| Variable | Description | Required |
|---|---|---|
| `VITE_FIREBASE_API_KEY` | Firebase project API key | No* |
| `VITE_FIREBASE_AUTH_DOMAIN` | Firebase Auth domain | No* |
| `VITE_FIREBASE_PROJECT_ID` | Firebase project ID | No* |
| `VITE_FIREBASE_STORAGE_BUCKET` | Firebase storage bucket URL | No* |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | Firebase Cloud Messaging sender ID | No* |
| `VITE_FIREBASE_APP_ID` | Firebase application ID | No* |
| `VITE_GEMINI_API_KEY` | Google Gemini API key for AI wellness tips | No* |

> \* **All Firebase variables are optional.** Without them, MindBoard automatically falls back to local storage mock mode. The Gemini API key is optional — without it, AI-generated wellness tips will be unavailable, but all other features work normally.
